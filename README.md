# The Carbon Detective 🕵️

A gamified home & classroom/office **carbon audit**, themed as a detective case file.
Pick a scene, sweep each zone for wasted electricity, fuel and water, and close the case
with a full audit report — CO₂, ₹ savings, prime suspects, and a transparent "show the
working" ledger.

Originally a single 1,036-line HTML file; rebuilt here as a **Vite + React + TypeScript**
application with the carbon math extracted into a pure, unit-tested core.

## AI Case Analyst (Claude)

After an audit closes, the **AI Case Analyst** turns the structured findings into a
personalised, prioritised action plan — streamed in live. It is powered by **Claude
(`claude-opus-4-8`)** through a serverless proxy (`api/analyze.ts`), and is designed to
the following guarantees:

- **Key stays server-side.** The `ANTHROPIC_API_KEY` lives only in the serverless
  function's environment. The Anthropic SDK is never bundled into the browser (verified in
  CI by inspecting `dist/`).
- **No PII leaves the device.** Only anonymous, already-computed audit numbers and the
  clue ids the user saw are sent — never the detective's name, case number, or timestamps.
  The boundary is enforced in one place (`src/core/aiBrief.ts`) and unit-tested.
- **Untrusted input is validated and bounded** server-side (`parseAnalystRequest`) — types,
  string lengths, numeric ranges, and finding count are all capped.
- **Graceful degradation.** With no key (or on any network/rate-limit error) the app falls
  back to a deterministic **on-device brief**, so the static SPA has _no hard backend
  dependency_ and works fully offline. This path is covered by an e2e test.
- **Responsible-AI disclosure.** The panel labels output as AI-generated, repeats that all
  figures are typical annual estimates (not advice), and the system prompt forbids invented
  figures or fabricated sources.

Set `ANTHROPIC_API_KEY` in your serverless host (see `.env.example`); `vercel.json` ships a
strict Content-Security-Policy and security headers. Without a key, everything else runs.

## Quick start

```bash
npm install
npm run dev            # start the dev server (Vite)
npm run lint           # ESLint (typescript-eslint + react-hooks)
npm run format:check   # Prettier formatting check
npm test               # unit tests (Vitest)
npm run test:coverage  # unit tests + V8 coverage (gated ≥95%, currently 100%)
npm run test:e2e       # end-to-end + axe a11y tests (Playwright) on the prod build
npm run build          # type-check + production build to dist/
npm run preview        # serve the production build
```

> First e2e run: `npx playwright install chromium` to fetch the browser.

CI (`.github/workflows/ci.yml`) runs, on every push and PR to `main`:
`npm audit` (fails on high/critical) → lint → format check → type-check →
unit + coverage → build → e2e/a11y.

## Responsible AI, privacy & transparency

This is a **deterministic, on-device** tool — and that is a deliberate, responsible choice.

- **No data leaves the device.** All state lives in `localStorage`; there are no servers,
  accounts, cookies, analytics or third-party trackers. Users can **export** a case file
  (`Download case file (.json)`) to keep their data, or open a new case to discard it. The
  only external request is the web-font stylesheet (locked down by CSP).
- **No black-box AI, by design.** Every number is a transparent, inspectable calculation
  from published emission factors — surfaced in full in the report's "Show the working"
  ledger and cited in the Field Manual (CEA, BEE, IPCC, EPA, IEA, GHG Protocol, DEFRA).
  Nothing is generated by a model, so nothing can be hallucinated or silently biased.
  Inspector Hoot is a scripted guide, not a decision-maker.
- **Honest about uncertainty.** Figures are labelled **typical annual estimates**, not meter
  readings, everywhere they appear. Assumptions live in `src/core/factors.ts`.
- **Humane UX.** The app nudges action, not anxiety: no dark patterns, no guilt-tripping.
  Motion, the flashlight cursor and confetti all respect `prefers-reduced-motion`, and the
  whole flow is keyboard-navigable with a skip link and visible focus.

The in-app **Responsible use** panel (`ResponsibleNote.tsx`) states all of this to users.

## Security

- **Hardening headers** ship via `vercel.json` (and `public/_headers` for Netlify/Cloudflare):
  a strict `Content-Security-Policy` (`default-src 'self'`, fonts allow-listed, `object-src
'none'`, `frame-ancestors 'none'`), `X-Content-Type-Options: nosniff`, `X-Frame-Options:
DENY`, `Referrer-Policy: no-referrer`, a locked-down `Permissions-Policy`, and HSTS.
- **No injection surface.** User input (the detective name) renders as escaped JSX text;
  there is no `dangerouslySetInnerHTML`, no `eval`, no remote code.
- **Supply chain.** `npm audit` is clean and runs in CI at `--audit-level=high`.
- **Resilient storage.** All `localStorage` access is guarded, so private-mode / quota /
  blocked-storage failures degrade silently instead of crashing.

## Architecture & scalability

- **Static, stateless SPA.** The build is plain HTML/CSS/JS served from a CDN with
  immutable, fingerprinted assets (`Cache-Control: immutable`) and a no-cache `index.html`.
  There is no backend, no database and no per-user server state, so it scales horizontally
  to any number of users at near-zero marginal cost.
- **Data-driven content.** Adding an audit clue is a single typed object in
  `src/core/clues.ts`; zones, the scene map, progress and the report all derive from it. New
  scenes/regions/tariffs extend the same data model without touching UI code.
- **Clean layering.** A pure, side-effect-free `core/` (100% unit-tested) sits under thin
  React components and small `lib/` adapters — easy to reuse, port, or wrap in other shells.

## Quality gates

- **Tests:** Vitest unit + component tests with **100% V8 coverage** of `core/` and `lib/`
  (gate ≥95%); Playwright e2e covering the full flow, persistence, and reload; automated
  **axe** WCAG 2.1 A/AA scans of the briefing and report (zero violations).
- **Static analysis:** ESLint (`typescript-eslint` + `react-hooks`) and Prettier, both
  enforced in CI; `tsc` in `strict` mode with `noUnusedLocals`/`noUnusedParameters`.

## What changed from the original (review fixes)

1. **Self-XSS removed.** The detective-name field was previously concatenated into the
   report via `innerHTML`. In React it flows through JSX as text and is escaped by
   construction — a name like `<img src=x onerror=…>` is shown literally.
2. **Badge contrast fixed.** The `MODERATE` and `LOW` severity badges were darkened to clear
   WCAG AA (4.5:1) for small text on the paper background.
3. **Testable carbon core.** All audit math lives in `src/core/audit.ts` as pure functions.
